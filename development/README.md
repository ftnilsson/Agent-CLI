# Development Skills

Universal software development skills that apply across every language, framework, and domain — from web and mobile to backend, embedded, data, and game development.

These are the **meta-skills** that compound over an entire career. A developer who masters these five will be effective in any tech stack, on any team, in any codebase.

## Skills

| # | Skill | Summary |
|---|-------|---------|
| 01 | [Architecture & System Design](skills/01-architecture-and-system-design/skill.md) | Decompose problems, manage complexity, and design systems that survive change |
| 02 | [Version Control & Git](skills/02-version-control-and-git/skill.md) | Collaborate effectively through branching strategies, commit hygiene, and code review |
| 03 | [Debugging & Problem Solving](skills/03-debugging-and-problem-solving/skill.md) | Apply the scientific method to find and fix bugs systematically |
| 04 | [Code Quality & Refactoring](skills/04-code-quality-and-refactoring/skill.md) | Write clean, maintainable code and improve existing code safely |
| 05 | [Testing Strategies](skills/05-testing-strategies/skill.md) | Build the safety net that gives you the confidence to ship and change code |

## How These Skills Relate

```
                    ┌──────────────────────────┐
                    │  01 Architecture & Design │  ← Guides all decisions
                    └────────────┬─────────────┘
                                 │
              ┌──────────────────┼──────────────────┐

   ┌──────────────────┐ ┌───────────────┐ ┌──────────────────┐

   │ & Refactoring    │ │  Strategies   │ │ Control & Git    │
   └────────┬─────────┘ └───────┬───────┘ └──────────────────┘
            │                   │
            └─────────┬─────────┘
                      ▼

           │ 03 Debugging &       │  ← The daily reality
           │ Problem Solving      │
           └──────────────────────┘
```

Architecture informs how you structure code. Code quality and testing protect that structure. Version control lets you collaborate on it. And debugging is what you do when reality diverges from intention — which is most of the time.
